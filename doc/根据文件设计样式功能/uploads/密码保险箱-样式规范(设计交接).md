# 密码保险箱 · 样式规范（设计交接）

> 用途：把「密码保险箱」界面的**现有样式逻辑 + 技术约束**交给设计侧优化。
> 功能已实现并跑通，本次只调**视觉/布局/交互质感**，不改数据结构与业务逻辑。
> 参考实现：`UmbraPC/src/features/vault/VaultApp.tsx`（React 组件）；可交互旧预览：`doc/密码保险箱-页面预览.html`。

---

## 0. 优化目标

- 核心气质：**安全、可信、克制、好用**。密码类工具要让人第一眼觉得「靠谱、专业」。
- 信息密度适中，长期高频使用不累。
- 深浅两色都要好看（跟随系统）。
- 在不引入重型 UI 库的前提下，把间距/层级/圆角/对比度/空态/微交互打磨到位。

---

## 1. 技术约束（实现侧，务必遵守）

1. **技术栈**：React + Tailwind（**只用 Tailwind 核心工具类**，可用 `[任意值]` 语法如 `px-[10px]`、`text-[13px]`）。不接入 shadcn/MUI 等组件库。
2. **颜色只用「设计令牌」CSS 变量**（见 §2），通过 Tailwind 的 `bg-card / border-border / text-muted / bg-orange / text-danger / bg-orange/10` 等类使用。**不要写死 16 进制色**（除非是画廊占位色这类装饰），以便深浅色自动切换。
3. **独立窗口**：这是一个单独的 Electron 窗口（`vault.html`），有原生标题栏；内容区自适应窗口，最小 900×600，默认 1080×720。
4. **图标现状是 emoji**（🔐🔑💳🪪📝 等）。可以提出更专业的图标方案，但若要换成矢量图标集需另行接入图标源——请在稿子里注明「emoji 保留」或「建议图标集 X」，实现侧再定。
5. **字体**：系统字体栈 `-apple-system, "SF Pro Text", system-ui, "Segoe UI", Roboto`，基准 14px。等宽用于密码/账号：`ui-monospace, Menlo, monospace`。
6. 不用外部图片资源；插画/图形请用内联 SVG 或纯 CSS。
7. 交付物最好是**更新后的 Tailwind 类**（可直接替换组件里的 className），或高保真视觉稿 + 标注（间距/字号/色值映射到令牌）。

## 4. 屏与组件现状（含当前样式，供优化参照）

### 4.1 初始化 / 解锁页（`Setup` / `Unlock`）
- 全屏 `bg-bg`，居中列（w-360）。大标题 `text-[20px] font-semibold` + 副标题 `text-[12.5px] text-muted`。
- 输入框 `.inp`：`border border-border bg-card rounded-[12px] px-[14px] py-[11px] text-[14px]`，聚焦 `border-orange`。
- 主按钮 `.btn-primary`：`bg-orange text-white rounded-[12px] py-[11px] text-[14px] font-semibold`（整宽）。
- **Secret Key 展示**：等宽大字 `font-mono text-[17px] tracking-wide`，`bg-bg border rounded-[12px]` 卡片，`select-all` 便于复制。强调「只显示一次、务必保存」。
- 优化点：这几屏是「安全第一印象」，希望更有仪式感/信任感（可加锁形图标、层次、微动效）。

### 4.2 顶栏
- 高 52，`bg-card border-b`。左品牌：22×22 圆角橙块 emoji + 「保险箱」。
- 身份切换：`border rounded-[10px] px-[10px] py-[5px]` 胶囊，点开下拉（`bg-card border rounded-xl shadow-2xl`，项 hover `bg-orange/10`）。
- 右：`＋添加记录`（橙实心）+`🔒锁定`（描边）。

### 4.3 左侧类型（可编辑文件夹）
- `w-196 bg-card border-r`；项 `px-[10px] py-2 rounded-[9px] text-[13px]`；选中 `bg-orange/10 text-orange font-semibold`。
- 固定项：🗂️全部、⭐收藏；其下是用户类型列表，右侧显示计数；**hover 时计数换成 ✎改名 / 🗑删除**。
- 改名进入行内 `input`（`border-orange`）。底部「＋ 新建类型」虚线按钮。

### 4.4 中间列表
- `w-290 border-r`。顶部搜索框（整宽，`bg-bg border rounded-[10px]`）。
- 记录卡片：34×34 圆角图标 + 标题 `text-[13.5px] font-semibold` + 副标题（首个账号或类型名）`text-[11.5px] text-muted`；收藏右侧 ⭐。选中 `bg-orange/10`。
- 空态：居中 `text-muted`「没有匹配的记录」。

### 4.5 右侧详情（控件流）—— 重点
一条记录 = 顶部信息 + 竖向堆叠的**控件卡片** + 底部「＋添加控件」。

**顶部信息**：50×50 圆角图标（`bg-orange/10`）+ 标题 `text-[19px] font-semibold` + 类型行（`text-muted` + 「移动到…」小按钮下拉）+ 收藏 ☆/⭐ + 「✏️编辑 / 保存」。

**控件卡片（`BlockCard`）**：`border rounded-[14px] bg-card mb-3`。
- 卡头：`bg-bg/40 border-b`，左「类型标签」胶囊（`text-orange bg-orange/10 rounded-md px-2`）+ 标签名；编辑态右侧出现 ↑↓/🗑。
- 卡体各类型：
  - **账号 account**：用户名行 / 密码行(打码•、👁 显示、📋 复制) / 网址行；编辑态密码行有 🎲 生成器；可勾「含 2FA」。
  - **密文 secret**：单个打码行 + 👁 + 📋 + 🎲。
  - **文本 text**：查看=多行段落；编辑=`textarea`。
  - **字段 field**：一行 值 + 📋。
  - **图片 images / 文件 files（`Gallery`）**：图片=120×80 圆角缩略图网格（`object-cover`）+ 编辑态右上 ✕ 删除 + 末尾虚线「＋添加图片」；文件=行式（📄 名称 + ⬇️导出 + ✕）。
- 密码/账号值用等宽字体；复制后 toast 提示。

**底部**：编辑态「＋ 添加控件」虚线块 → 弹 6 类控件按钮；一行安全说明 `🔒 整条已 AES-256-GCM 加密 · 密码/密文不进搜索`；编辑态右侧「删除记录」。

### 4.6 通用
- **Toast**：底部居中黑胶囊 `bg-[#17130f] text-white rounded-full`，1.4s 自动消失。
- **弹层**（身份/移动/加控件菜单）：`bg-card border rounded-xl shadow-2xl`，项 hover `bg-orange/10`。
- **打码**：密码默认 `•` × 8–12；👁 切换明文；🙈 收起。

## 5. 交互状态清单（请一并给态）

- 悬浮 hover：列表项、类型项（露出 ✎🗑）、控件卡（露出 ↑↓🗑）、可点图标（→ 变橙）。
- 选中 selected：类型/记录/身份（`bg-orange/10`）。
- 编辑态 vs 查看态：字段变输入框、出现增删排序、底部出现「添加控件/删除记录」。
- 空态：无记录、画廊为空、无搜索结果。
- 焦点 focus：输入 `border-orange`。
- 加载/解密：图片解密中占位文案。
- 危险操作：删除类/删记录/删附件用 `text-danger`。

---

## 6. 自由度：能改 / 别动

**可自由优化**：所有视觉——配色映射、间距节奏、圆角、阴影、字号层级、卡片/胶囊样式、图标风格建议、空态与微交互、三栏宽度、控件卡片布局与分组、解锁页的仪式感。

**请勿改动（否则要重接逻辑）**：
- 组件的**数据流与 props / 函数名**、`window.umbraVault` 的 IPC 调用；
- 六种控件的**类型标识**（account/secret/text/field/images/files）与其字段（username/password/url/otp、value、atts）；
- 三层结构（身份库 Vault × 类型 Type × 记录 Item + 控件 Block）；
- 「密码/密文不进搜索、默认打码」这条安全规则。

> 换句话说：**结构与交互契约不变，换一身好看的皮**。产出优化后的 Tailwind 类（或标注清晰的视觉稿 + 令牌映射），实现侧照着替换 className 即可。
